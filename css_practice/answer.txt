1. Разница между классом и ID:
   - Класс (.class) - специфичность 0,1,0, можно использовать много раз
   - ID (#id) - специфичность 1,0,0, должен быть уникальным

2. Каскад в CSS:
   При конфликте правил применяются в порядке:
   1. !important
   2. Специфичность селектора
   3. Порядок в коде

3. Расчет специфичности:
   - ul li.important = 0,1,2 (2 элемента + 1 класс)
   - #site-title = 1,0,0 (1 ID)
   - .content > .featured-post = 0,2,0 (2 класса)

4. Наследуемые свойства:
   Наследуются: color, font-family, line-height
   Не наследуются: background, border, margin, padding

5. Дочерний селектор >:
   Выбирает только непосредственных потомков
   .parent > .child выберет только прямых детей

6. Псевдоклассы:
   :hover - стили при наведении
   :nth-child() - выбор по порядковому номеру
   Полезны для интерактивности

7. !important:
   Повышает приоритет правила
   Использовать осторожно, чтобы не нарушить каскад

8. ::before и ::after:
   Создают псевдоэлементы для декоративного контента
   Могут содержать текст, иконки

9. При одинаковой специфичности:
   Применяется последнее правило в коде

10. Инструменты разработчика:
    Показывают примененные стили, специфичность, помогают в отладке

CSS-переменные: добавлены в :root, используются для цветов и размеров

Анимации: созданы fadeIn и bounce через @keyframes

Медиа-запросы: добавлена адаптивность для мобильных устройств

currentColor: использован для наследования цвета в границах
