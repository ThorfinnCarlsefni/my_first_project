# Ответы на вопросы по Flexbox

1. **Какое основное свойство CSS включает Flexbox и что происходит с дочерними элементами после его применения?**
   - Основное свойство: `display: flex`. После его применения дочерние элементы становятся flex-элементами и выстраиваются вдоль основной оси.

2. **В чем разница между justify-content и align-items?**
   - `justify-content` выравнивает элементы по основной оси
   - `align-items` выравнивает элементы по поперечной оси
   - Пример: `justify-content: center` центрирует по горизонтали, `align-items: center` - по вертикали

3. **Что делает свойство flex-direction и какие значения может принимать?**
   - Определяет направление основной оси. Значения: `row`, `row-reverse`, `column`, `column-reverse`

4. **Объясните значение трех свойств в сокращенной записи flex: 1**
   - `flex: 1` эквивалентно `flex: 1 1 0`
   - `flex-grow: 1` - элемент может увеличиваться
   - `flex-shrink: 1` - элемент может уменьшаться
   - `flex-basis: 0` - базовый размер равен 0

5. **Как работает свойство flex-wrap и в каких случаях его следует использовать?**
   - Определяет, могут ли элементы переноситься на новую строку. Используется когда элементы не помещаются в контейнере.

6. **Чем отличается align-content от align-items?**
   - `align-items` выравнивает отдельные элементы по поперечной оси
   - `align-content` выравнивает строки flex-элементов (работает только при `flex-wrap: wrap`)

7. **Как свойство order влияет на отображение элементов?**
   - Изменяет порядок отображения элементов. Полезно для изменения порядка без изменения HTML-структуры.

8. **Что такое "осевая линия" (main axis) и поперечная ось (cross axis)?**
   - Основная ось - направление, заданное `flex-direction`
   - Поперечная ось - перпендикулярное направление

9. **Как создать адаптивный макет с помощью Flexbox без медиа-запросов?**
   - Использовать `flex-wrap: wrap` и `flex-basis` для автоматического переноса элементов

10. **Какие преимущества дает использование gap вместо margins?**
    - `gap` создает равные промежутки между всеми элементами
    - Упрощает код, не нужно учитывать margin-collapse
    - Легче поддерживать одинаковые расстояния
