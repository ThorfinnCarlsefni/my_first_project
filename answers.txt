Ответы на вопросы для самоконтроля

1. Какие три основные области работы с файлами в Git и чем они отличаются?

Рабочая папка (Working Directory) - текущее состояние файлов в проекте, где вы вносите изменения.
Индекс (Staging Area) - промежуточная область, куда добавляются файлы перед коммитом.
Репозиторий (Repository) - база данных Git, где хранятся все коммиты и история проекта.

2. Что происходит при выполнении команд:

- git add - добавляет изменения из рабочей папки в индекс
- git commit - создает коммит из проиндексированных изменений
- git push - отправляет локальные коммиты в удаленный репозиторий

3. Почему важно писать информативные сообщения к коммитам?

Информативные сообщения помогают понимать историю изменений, находить нужные коммиты, сотрудничать в команде и облегчать отладку.

4. Чем отличается git add имя_файла от git add .?

git add имя_файла - добавляет только указанный файл
git add . - добавляет все измененные файлы в текущей директории

5. Что показывает команда git status?

Показывает: измененные файлы, файлы в индексе, неотслеживаемые файлы и состояние ветки.

6. Для чего нужна команда git remote add origin URL и что означает слово "origin"?

Связывает локальный репозиторий с удаленным. "origin" - стандартное имя для удаленного репозитория.

7. Зачем нужен флаг -u в команде git push -u origin main?

Устанавливает связь между локальной и удаленной веткой, чтобы потом использовать просто git push.

8. Что такое ветка по умолчанию и какая у вас создалась?

Ветка по умолчанию - основная ветка разработки. У меня создалась ветка main.

9. Как просмотреть историю коммитов?

Командой git log. Видно: хеш коммита, автора, дату и сообщение коммита.

10. Что произойдет, если сделать изменения в файле, но не выполнить git add перед git commit?

Изменения не попадут в коммит, так как они не были добавлены в индекс.
